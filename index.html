Tentang Proyek

Apa itu CMMS Smart Factory?

Computerized Maintenance Management System (CMMS) adalah platform digital terpadu untuk mengelola seluruh aktivitas pemeliharaan mesin — dari laporan kerusakan hingga prediksi kerusakan sebelum terjadi.

Masalah yang Diselesaikan

  • Laporan kerusakan mesin hanya via WhatsApp — tidak ada jejak audit yang jelas
  • Status perbaikan tidak bisa dipantau real-time oleh SPV dan Manajer
  • Tidak ada histori kerusakan tersimpan untuk analisis pola dan tren
  • Alur approval Purchase Request (spare part) tidak terdokumentasi
  • Preventive Maintenance tidak terjadwal otomatis — bergantung pada ingatan manusia

Fase 1 — MVP & Fondasi Inti

Menggantikan laporan WhatsApp dan Infinity Helpdesk dengan sistem tiket digital berbasis QR Code

Bulan 1–2 · Prioritas Utama
🎯 Tujuan Utama
  • 100% laporan kerusakan masuk via sistem tiket digital
  • Nol laporan kerusakan via WhatsApp
  • SPV bisa memonitor semua tiket secara real-time
  • Audit trail lengkap untuk setiap insiden mesin
📦 Fitur yang Dibangun
  • Generator QR Code unik per mesin (dipasang sebagai stiker)
  • Form laporan otomatis muncul saat QR di-scan (tanpa download app)
  • Dashboard tiket untuk SPV: assign, update status, pantau progres
  • Manajemen pengguna: Teknisi, SPV, Manajer, Direktur
  • Riwayat tiket per mesin untuk analisis ke depan

Fase 2 — Automasi & Reporting

Laporan otomatis ke Manajer, scheduler Preventive Maintenance, dan notifikasi tiket overdue

Bulan 3–5 · Lanjutan Fase 1
🤖 Fitur Automasi Baru
  • Laporan PDF otomatis (ringkasan tiket mingguan/bulanan) dikirim via email ke Manajer
  • Export data ke format Excel untuk analisis lanjutan
  • Scheduler otomatis: buat tiket PM setiap tanggal 1 berdasarkan daftar mesin
  • Notifikasi email jika tiket melewati batas waktu (overdue)
  • Dashboard laporan dengan grafik tren kerusakan per mesin / area
⚡ Mengapa Celery + Redis?

Tanpa background worker, saat server diminta membuat laporan PDF tebal atau mengirim email ke banyak penerima sekaligus, server akan "hang" dan pengguna lain tidak bisa mengakses sistem.

Solusinya:

Celery memindahkan pekerjaan berat ke antrian yang diproses secara asinkron di background — server tetap responsif untuk pengguna lain.

Fase 3 — Supply Chain & Procurement

Manajemen stok spare part dengan alert otomatis dan alur approval Purchase Request multi-level

Bulan 8–13 · Integrasi Pengadaan
📦 Fitur Supply Chain
  • Inventori spare part terintegrasi — stok, lokasi gudang, minimum buffer
  • Auto-alert: PostgreSQL Trigger otomatis jika stok turun di bawah batas minimum
  • Modul PR dengan alur approval multi-level: Teknisi → SPV → Manajer → Direktur
  • Generate dokumen PO (Purchase Order) PDF otomatis setelah disetujui
  • Upload foto kerusakan & bukti tersimpan di cloud (Cloudflare R2 / AWS S3)
💡 Logika State Machine

Setiap PR hanya bisa berpindah ke status berikutnya setelah disetujui pejabat berwenang.

Sistem mencatat:
  • Siapa yang menyetujui
  • Kapan disetujui (timestamp)
  • Catatan jika ada penolakan

Efek: Direktur hanya melihat PR yang sudah disetujui SPV dan Manajer — tidak kelebihan beban persetujuan.

⚡ Auto-Alert Stok Kritis (PostgreSQL Trigger)

Saat stok V-Belt, Oil Filter, atau spare part kritis lain turun di bawah batas minimum — sistem secara otomatis membuat draft PR dan mengirim notifikasi ke SPV untuk tindak lanjut.

Fase 4 — Enterprise, Mobile & AI

Multi-pabrik, Predictive Maintenance berbasis AI, dan akses offline melalui Mobile App native

Tahun 2–3 · Transformasi Enterprise
Contoh Notifikasi AI Predictive Maintenance
🔮 Prediksi Kerusakan — Mesin Pompa #3

Mesin Pompa #3 biasanya mengalami kerusakan setiap 40 hari. Namun berdasarkan data sensor suhu 2 minggu terakhir yang konsisten di atas normal, model memprediksi kemungkinan kerusakan dalam 15 hari ke depan.

Rekomendasi: Lakukan inspeksi dalam 7 hari. — Notifikasi dikirim otomatis ke SPV dan Manajer.
Model: Scikit-learn + Prophet (Python) · Akurasi target ≥80% · Data: histori jutaan titik kerusakan
📱 Mobile App Offline-First
  • Flutter / React Native — tersedia di Google Play Store & App Store
  • Teknisi bisa update tiket tanpa sinyal WiFi
  • Data sinkron otomatis saat koneksi tersedia kembali
  • JWT Token untuk autentikasi aman di mobile
🏭 Multi-Tenant & Skalabilitas
  • Satu sistem untuk banyak pabrik/cabang dengan isolasi data per tenant
  • Docker + Kubernetes di AWS / GCP — auto-scaling saat pengguna meningkat
  • PostgreSQL Master-Slave replication untuk keandalan tinggi
  • WhatsApp Business API sebagai channel notifikasi tambahan

🐍 Mengapa Python Sejak Fase 1?

Pilihan Python bukan hanya untuk kemudahan pengembangan awal. Saat proyek mencapai Fase 4, tim Data Scientist tidak perlu mempelajari bahasa baru untuk membangun model AI. Library Scikit-learn, Prophet, dan Pandas dapat langsung digunakan dalam ekosistem yang sama — mempersingkat waktu pengembangan AI secara signifikan.

Risiko Level Strategi Mitigasi
Adopsi Pengguna
Teknisi tidak terbiasa sistem digital		 🔴 Tinggi Pelatihan singkat, antarmuka mobile-friendly, QR Code menghilangkan kebutuhan login manual

Render.com (gratis) dapat downtime		 🟡 Sedang Fase 1 masih bisa fallback ke WA jika sistem down; upgrade ke plan berbayar sebelum Fase 2
Kehilangan Data

Overscoping
Terlalu banyak fitur sekaligus		 🟡 Sedang Setiap fase harus deliver MVP-nya sebelum memulai fase berikutnya
